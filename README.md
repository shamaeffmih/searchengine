# searchengine
### Поисковый движок по сайтам
Перед вами приложение, которое позволяет индексировать страницы и осуществлять по ним быстрый поиск. Движок разработан на фреймворке Spring Boot, состоит из frontend и backend составлящих.


##### В данном проекте:

* реализовано многопоточное приложение, которое обходит все страницы сайта начиная с главной;
* использована библиотека лемматизации слов;
* реализована система индексации страниц сайта — система, которая позволяет подсчитывать слова на страницах сайта и по поисковому запросу определяет наиболее релевантные (соответствующие поисковому запросу) страницы;
* реализована система поиска информации с использованием созданного поискового индекса;
* использовано веб-приложение, которое индексирует несколько сайтов.

##### Стек технологий:
- Spring Boot
- JDK (java 17)
- Spring Data JPA + Hibernate
- MySQL
- Apache lucene morphology
- Ломбок

#### Описание веб-интерфейса
Веб-интерфейс (frontend-составляющая) проекта представляет собой
одну веб-страницу с тремя вкладками:

● Dashboard

Эта вкладка открывается по умолчанию. На ней отображается общая статистика по всем сайтам, а также детальная статистика и статус по каждому из сайтов (статистика, получаемая по запросу /api/statistics).

● Management

На этой вкладке находятся инструменты управления поисковым движком — запуск и остановка полной индексации (переиндексации), а также возможность добавить (обновить) отдельную страницу по ссылке.

● Search

Эта страница предназначена для тестирования поискового движка. На ней находится поле поиска, выпадающий список с выбором сайта для поиска, а при нажатии на кнопку «Найти» выводятся
результаты поиска (по API-запросу /api/search).

#### Структура базы данных
**site** — информация о сайтах и статусах их индексации

- id
- status
- status_time
- last_error
- url
- name

**page** — проиндексированные страницы сайта
 - id
 - site_id
 - path
 - code
 - content
 
 **lemma** — леммы, встречающиеся в текстах
- id
- site_id
- lemma
- frequency

**index** — поисковый индекс
- id
- page_id
- lemma_id
- rank

#### Спецификация API

***Запуск полной индексации*** — GET /api/startIndexing

Метод запускает полную индексацию всех сайтов или полную переиндексацию, если они уже проиндексированы. Если в настоящий момент индексация или переиндексация уже
запущена, метод возвращает соответствующее сообщение об ошибке.

***Остановка текущей индексации*** — GET /api/stopIndexing

Метод останавливает текущий процесс индексации (переиндексации). Если в настоящий момент индексация или переиндексация не происходит, метод возвращает соответствующее сообщение об ошибке.

***Добавление или обновление отдельной страницы*** — POST /api/indexPage

Метод добавляет в индекс или обновляет отдельную страницу, адрес которой передан в параметре. Если адрес страницы передан неверно, метод возвращает соответствующую ошибку.

*Параметры:*
- url — адрес страницы, которую нужно переиндексировать.


***Статистика*** — GET /api/statistics

Метод возвращает статистику и другую служебную информацию о состоянии поисковых индексов и самого движка.

***Получение данных по поисковому запросу*** — GET /api/search

Метод осуществляет поиск страниц по переданному поисковому запросу. Чтобы выводить результаты порционно задан сдвиг от начала списка результатов и количество результатов, которое необходимо вывести. В ответе выводится общее количество результатов и массив с результатами поиска. Каждый результат — это объект, содержащий свойства результата поиска. Если поисковый запрос не задан или ещё нет готового индекса метод возвращает соответствующую ошибку, понятную обычному пользователю.

*Параметры:*
- query — поисковый запрос;
- site — сайт, по которому осуществлять поиск (если не задан, поиск происходит по всем проиндексированным сайтам)
 - offset — сдвиг от 0 для постраничного вывода
 - limit — количество результатов, которое выводится

#### Инструкция по запуску проекта

***настроить файл конфигурации application.yaml***
1. описать подключение к БД в разделе *spring:*

````
spring:
  datasource:
    username: ****
    password: ****
    url: jdbc:mysql://localhost:3306/search_engine?useSSL=false&requireSSL=false&allowPublicKeyRetrieval=true
````
2. задать список сайтов в разделе *indexing-settings:*

````
indexing-settings:
  sites:
    - url: https://tchaikovskyhome.ru/
      name: Музей П. И. Чайковского
    - url: https://www.playback.ru/
      name: Плей Бэк
    - url: https://radiomv.ru/
      name: Милицейское радио
````
3. запустить приложение или в среде разработки или через командную строку, предварительно скомпилировав *.jar файл проекта (application.yaml должен быть отдельно)
4. после запуска приложения в веб-браузере ввести: ***localhost:8080***
